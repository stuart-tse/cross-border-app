🚗 CrossBorder Dashboard Design System

Tesla-inspired dashboard interface with light pink (#FF69B4) accents for multi-role authentication system

🎯 Problem Statement

Current Issue: Users are seeing "Access Denied" after logging in instead of being redirected to a proper dashboard.

Requirements Summary

  • Design Tesla-inspired dashboard layout with light pink (#FF69B4) accents
  • Create role-specific dashboards for CLIENT, DRIVER, BLOG_EDITOR, and ADMIN users
  • Include relevant widgets, quick actions, and data visualization for each role
  • Responsive design following existing design system
  • Update navigation to show "Dashboard" instead of "Login" when authenticated

🎨 Design Philosophy

Tesla-Inspired Principles

  • Minimalism: Clean, uncluttered interfaces with purposeful white space
  • Premium Quality: High-end materials, precise typography, sophisticated color usage
  • Innovation Focus: Forward-thinking design suggesting cutting-edge technology
  • Functional Beauty: Every element serves both aesthetic and functional purposes
  • Seamless Experience: Intuitive interactions with subtle, meaningful animations

Breakpoint Strategy

Mobile (320px - 767px)

  • Sidebar collapses to bottom navigation
  • Widget grid becomes single column
  • Stats display in 2x2 grid
  • Simplified quick actions

Tablet (768px - 1023px)

  • Sidebar becomes slide-out drawer
  • Widget grid 1:1 ratio
  • Stats in horizontal row
  • Touch-optimized controls

Desktop (1024px+)

  • Full sidebar navigation
  • 2:1 widget grid ratio
  • All dashboard features
  • Hover interactions

interface DashboardLayoutProps { userRole: UserType; children: React.ReactNode; sidebarItems: NavItem[]; headerActions?: React.ReactNode; } export const DashboardLayout: React.FC<DashboardLayoutProps> = ({ userRole, children, sidebarItems, headerActions }) => { return ( <div className="dashboard-container"> <DashboardSidebar items={sidebarItems} userRole={userRole} className="dashboard-sidebar" /> <main className="dashboard-main"> <DashboardHeader userRole={userRole} actions={headerActions} /> <div className="dashboard-content"> {children} </div> </main> </div> ); };

Widget System

interface WidgetProps { title: string; icon?: string; loading?: boolean; error?: string; className?: string; actions?: React.ReactNode; children: React.ReactNode; } export const Widget: React.FC<WidgetProps> = ({ title, icon, loading, error, className, actions, children }) => { if (loading) return <WidgetSkeleton />; if (error) return <WidgetError message={error} />; return ( <Card className={cn("widget-card", className)}> <WidgetHeader title={title} icon={icon} actions={actions} /> <WidgetContent>{children}</WidgetContent> </Card> ); };

Role-Based Dashboard Hook

export const useDashboard = (userRole: UserType) => { const [widgets, setWidgets] = useState<DashboardWidget[]>([]); const [loading, setLoading] = useState(true); const [error, setError] = useState<string | null>(null); useEffect(() => { const loadDashboardData = async () => { try { setLoading(true); const data = await fetchDashboardData(userRole); setWidgets(data.widgets); } catch (err) { setError(err.message); } finally { setLoading(false); } }; loadDashboardData(); }, [userRole]); return { widgets, loading, error }; };
